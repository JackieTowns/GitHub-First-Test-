To keep repo public when making so won't get authenication failure and make sure to have it make a read me file

  • dir : shows the list of folders within a directory

In the folder you want the reposity to be "git clone" and paste the URL from the repo. Shortcut doesn't work so rightclick to paste

Git Commands

git init : intalizes the folder you want git to use ie in this case the Bootcamp folder

git status : get the git status of the file

git add : Add the file you want to commit

git commit : add the commit use "git commit -m "add file X" " to add a message. If you don't use the -m a text editor will appear and same thing.

git add . : use for committing everything in that place at one time and then use git commit to commit the combined files

Tell us who you are error: git config --global user.name "enter PC name found under Windows > Computer > System Properties > Computer Name" git config --global user.email "enter email"

When you make new changes you first add it and then commit it. A two step proecess...

git config -- list : check your username and email

Git Push

git remote origin : should show the word origin

git remote -v : shows the particular URL and it should be github repo

git push origin master: to push to GitHub. You should first see it asking for your username of github followed by your password. The login box for Github pops up and you put it in.
